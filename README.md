# vibrogen

Имитатор сигналов вибродиагностики на Qt6/C++.

## Что реализовано

- Генерация и вывод в стандартное аудиоустройство трёх типов сигналов:
  - гармонический сигнал;
  - белый шум;
  - сигнал типового дефекта (модулированный по скорости движения).
- Сценарии изменения скорости подвижного состава:
  - постоянная скорость;
  - разгон;
  - торможение;
  - циклический профиль (остановка → разгон → равномерное движение → торможение).
- Интерфейс приложения:
  - строка меню;
  - панель управления воспроизведением (кнопка Старт/Стоп);
  - строка состояния;
  - таблица текущего набора сигналов с добавлением/удалением;
  - панель из трёх вкладок (осциллограмма, спектр, waterfall);
  - диалоги параметров сценария и характеристик узлов ходовой части.
- Поддержка сохранения/открытия проекта генерации сигналов в JSON.
- Базовая конфигурация узлов: ЭС2Г «Ласточка» (максимальная скорость 150 км/ч).

## Сборка

```bash
cmake -S . -B build
cmake --build build
```

## Сборка для установки на другом компьютере (с Qt-библиотеками)

Если просто скопировать бинарник `vibrogen`, на другом ПК часто появляется ошибка
о том, что не найдены Qt DLL/SO/DYLIB и плагины платформы. Для переносимой установки
собирайте и устанавливайте проект через `cmake --install`:

```bash
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build --config Release
cmake --install build --config Release --prefix "$(pwd)/dist"
```

После этого в `dist` будет приложение и необходимые Qt-зависимости
(через `windeployqt`/`macdeployqt` и соответствующие механизмы Qt6).

> Важно: указывайте **абсолютный** `--prefix`, иначе Qt deploy-скрипт может
> завершиться ошибкой вида `Given qt.conf path is not an absolute path`.

### Windows

- Запускайте сборку из *Qt Command Prompt* (или после `qtenv2.bat`), чтобы
  инструменты развёртывания Qt были в `PATH`.
- Готовую папку `dist` можно архивировать и переносить на другой компьютер.

### Linux

- На целевой машине должны быть совместимые системные зависимости
  (glibc, драйверы аудио/ALSA/PulseAudio и т.д.).
- Для максимальной переносимости обычно делают AppImage/пакеты дистрибутива,
  но минимальный рабочий вариант — папка из `cmake --install`.

## Платформы

Проект целится в Windows, Linux и Android (через Qt6).
